AngularJS单元测试

讲了Java的单元测试，基于Spring框架的单元测试，Robot自动化测试，现在就来讲一下基于AngularJS的JavaScript单元测试。

Karma和Jasmine

Karma是一个JS的测试运行器，Jasmine是一个JS的单元测试框架。
* 首先我们在工程根目录下创建npm的一个配置文件——package.json，这个配置文件填写所需的依赖组件，比如我们需要Karma和Jasmine，package.json示例代码如下。

  • 然后在工程目录下执行npm install来进行依赖组件的安装。
    PS:npm install会默认从国外网站下载，速度较慢，可以使用淘宝的npm镜像进行下载:npm install --registry=https://registry.npm.taobao.org。

  • 安装完成后，会在工程目录下看到多出了一个node_modules的目录，这个目录下面就是我们跑JS测试所需要的依赖组件。

JS测试文件目录结构

我们以Spring的工程目录为例，其实JS的单元测试文件跟Spring的Java代码或者xml文件一点关系都没有，可以随意放置，但为了结构统一，可以放到src/test这个目录下面，目录结构如下:

  • config目录放置Karma的配置文件，后面会讲Karma配置文件怎么写。
  • lib目录放置跑测试需要的js依赖文件，比如AnglarJS的文件等。
  • unit目录放置js测试文件。

这里介绍一下主要的几个配置项:

  • basePath: 执行JS单元测试的开始路径。
  • files: 执行JS单元测试所需JS文件的路径，包含测试文件本身的路径。
  • browsers: 跑测试的浏览器，这里是Chrome。

编写完配置文件后，在项目根目录下执行命令node_modules/karma/bin/karma start src/test/js/config/karma.unit.js就可以跑JS单元测试了，执行命令后会看到弹出Chrome浏览器窗口，在命令窗口可以看到单元测试的执行结果。执行结果示例如下所示:
